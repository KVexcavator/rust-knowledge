Итераторный паттерн позволяет поочередно выполнять работу с последова­тельностью элементов. Итератор отвечает за логику перебора каждого элемента и определяет, когда эта последовательность закончилась.
---
В Rust итераторы являются ленивыми, то есть не действуют, пока не вызове­те методы, которые потребляют итератор до его исчерпания.
---
Создание итератора
let v1 = vec![1, 2, 3];
let v1_iter = v1.iter();
После создания итератора можно использовать его различными способами, например с for(не берет во владение)
for val in v1_iter {
  println!("Получено: {}", val);
}
---
Типаж Iterator и метод next
Все итераторы реализуют типаж Iterator, определенный в стандартной библио­теке следующим образом:
pub trait Iterator {
  type Item;
  fn next(&mut self) -> Option<Self::Item>;
}
1. Iterator требует, чтобы был определен тип Item, который используется в качестве типа, возвращаемого из метода next
2. Iterator требует определения одного метода - next, который возвращает по одному элементу итератора, завернутому в Some, за раз, а когда перебор завершается, он возвращает None
#[test]
fn iterator_demonstration() {
  let v1 = vec![1, 2, 3];
  let mut v1_iter = v1.iter();

  assert_eq!(v1_iter.next(), Some(&1));
  assert_eq!(v1_iter.next(), Some(&2));
  assert_eq!(v1_iter.next(), Some(&3));
  assert_eq!(v1_iter.next(), None);
}
---
- iter создает итератор для неизменяемых ссылок
- into_iter создаёт итера­тор, который берет v1 во владение и возвращает обладаемые значения
- iter_mut перебирает изменя­емые ссылки
---
вызов next для итератора изменяет внутреннее состояние, которое итератор ис­пользует для отслеживания своей позиции в последовательности. Другими сло­вами, этот код потребляет, или расходует, итератор. Каждый вызов метода next съедает элемент из итератора
---
Методы, которые потребляют итератор
Вызывают next и называются «потребляющие адаптеры», потому что их вызов расходует итератор
Например sum, мере перебора он добавляет каждый элемент в промежуточную сумму и возвращает её по завершении итераций
#[test]
fn iterator_sum() {
  let v1 = vec![1, 2, 3];
  let v1_iter = v1.iter();
  let total: i32 = v1_iter.sum();
}
---
Методы, которые производят другие итераторы
"итераторные адапте­ры" — позволяют переделывать итераторы в разные виды 
Можете выстроить несколько вызовов итераторных адаптеров в цепь
Поскольку все итераторы являются ленивыми, то для того что­ бы получить результаты вызовов итераторных адаптеров, придется вызвать один из методов потребляющего адаптера
Пример: Вызов map для создания нового итератора и последующий вызов метода collect для потребления нового итератора и создания вектора
let v1: Vec<i32> = vec![1, 2, 3];
let v2: Vec<_> = v1.iter().map(|x| x + 1).collect();
assert_eq!(v2, vec![2, 3, 4]);
