Ничего отключать не нужно — тип по умолчанию НЕ Copy
---
Все типы Rust НЕ Copy,
кроме тех, для которых программист ЯВНО (или автоматически) разрешил Copy
```
некопируемость — это норма, а Copy - opt-in
---
2. Когда тип становится Copy
Тип является Copy, если выполняются оба условия:
1. Ты прграммист написал:
```rust
#[derive(Copy, Clone)]
```
2. Это примитив/стандартный тип который уже Copy
И при этом:
- все поля тоже Copy
- нет Drop
---
3. Самый простой способ сделать тип некопируемым
НИЧЕГО НЕ ДЕЛАТЬ
```rust
struct Token {
    id: u64,
}
```
Token НЕ Copy
```rust
let a = Token { id: 1 };
let b = a;      // move
// println!("{}", a.id); use of moved value
```
---
4. Запретить Copy, даже если поля Copy
Если поля Copy, но ты хочешь запретить копирование:
Вариант 1: не деривать Copy
```rust
#[derive(Clone)]
struct Token {
    id: u64,
}
```
Теперь:
- Clone - явно
- Copy нет
---
Вариант 2: добавить Drop (жёсткий запрет)
```rust
struct Token {
    id: u64,
}

impl Drop for Token {
    fn drop(&mut self) {
        println!("dropping {}", self.id);
    }
}
```
Теперь:
- Copy невозможен принципиально
- даже если бы ты захотел
Компилятор:
Copy types cannot implement Drop
---
5. Специальный приём: PhantomData
Если тип логически владеет ресурсом, но физически — нет:
```rust
use std::marker::PhantomData;

struct Handle {
    id: u32,
    _not_copy: PhantomData<Box<()>>,
}
```
Box<()> - не Copy
значит Handle — тоже не Copy
Используется в низкоуровневых API
---
6. Запретить Clone
Clone нельзя запретить напрямую, но:
- просто не реализуй Clone
- не пиши #[derive(Clone)]
```rust
struct Secret {
    key: u64,
}
```
Любая попытка:
```rust
let b = a.clone(); // ❌
```
---
7. Типичный паттерн: можно клонировать, но дорого
```rust
#[derive(Clone)]
struct BigData {
    buf: Vec<u8>,
}
```
- Clone есть
- Copy нет
копирование - явное
---
8. Частая ошибка новичков
```rust
#[derive(Clone)]
struct A { x: i32 }

let a = A { x: 1 };
let b = a; // move, НЕ clone
```
Clone никогда не вызывается автоматически
Копирование всегда либо:
- Copy(неявно)
- clone() (явно)
---
9. Таблица решений

| Хочу                        | Что сделать             |
| --------------------------- | ----------------------- |
| Тип некопируемый            | просто не derive Copy   |
| Запретить Copy жёстко       | реализовать Drop        |
| Разрешить явное копирование | Clone                   |
| Запретить и Copy, и Clone   | ничего не делать        |
| Логическое владение         | PhantomData<Box<()>>    |
---
10. Ментальная модель (очень важная)
Move - это передача владения, а не копирование
Copy - это исключение из правила
