В Rust **почти все типы — `Sized`**.
Sized означает размер типа известен компилятору на этапе компиляции
Примеры Sized:
```rust
i32
bool
struct A { x: i32 }
[u8; 32]
String
Vec<i32>
```
Компилятор точно знает:
сколько байт занимает тип
как положить его в стек
---
Типы, которые НЕ `Sized`
Некоторые типы **по определению не имеют фиксированного размера**:
Срезы
```rust
[u8]      // не Sized
str       // не Sized
```
Почему?
- длина известна только в runtime
- [u8; 10] — ok
- [u8] — неизвестно сколько байт
---
Trait objects
```rust
dyn Error     // не Sized
dyn Display   // не Sized
```
Почему?
разные реализации имеют разный размер
компилятор не знает, какой конкретно тип лежит внутри
---
Что будет, если попробовать использовать`?Sized` тип напрямую
```rust
fn f(x: dyn Error) { } // ❌
```
Компилятор -  я не знаю, сколько байт класть в стек
---
Как Rust вообще работает с !Sized типами
Через указатели
```rust
&[u8]
&str
&dyn Error
Box<dyn Error>
Rc<dyn Trait>
```

Все они — **fat pointer**:

```
&dyn Error
 ├─ ptr на данные
 └─ ptr на vtable
```

Размер указателя **известен**, даже если размер данных — нет.

---

## 5. Теперь к `?Sized` — что это вообще такое

`?Sized` читается как:

> «тип **может быть не Sized**»

По умолчанию в generic-параметрах Rust **НЕЯВНО** делает:

```rust
<T: Sized>
```

То есть:

```rust
fn f<T>(x: T) { }
```

на самом деле значит:

```rust
fn f<T: Sized>(x: T) { }
```

---

## 6. Зачем писать `T: ?Sized`

Чтобы **разрешить** передавать unsized типы.

Пример из std:

```rust
struct Box<T: ?Sized> {
    ptr: *mut T,
}
```

Если бы не `?Sized`:

```rust
Box<dyn Error> // ❌ невозможно
Box<[u8]>      // ❌ невозможно
```

---

## 7. Пример: функция, работающая с Sized и unsized

```rust
fn print<T: ?Sized + std::fmt::Display>(x: &T) {
    println!("{}", x);
}
```

Работает для:

```rust
print("hello");              // &str (unsized)
print(&123);                 // i32 (sized)
print(&vec![1, 2, 3]);       // Vec<i32>
```

Обрати внимание:

👉 `x: &T`, а не `x: T`
👉 потому что `T` может быть unsized

---

## 8. Почему `T: ?Sized` почти всегда идёт с `&T` или `Box<T>`

Потому что:

```rust
fn f<T: ?Sized>(x: T) { } // ❌
```

Нельзя положить `T` в стек, если он unsized.

А вот так — можно:

```rust
fn f<T: ?Sized>(x: &T) { } // ✅
```

---

## 9. Частый пример из стандартной библиотеки

```rust
impl<T: ?Sized> Deref for Box<T> {
    type Target = T;

    fn deref(&self) -> &T {
        unsafe { &*self.ptr }
    }
}
```

Вот почему `Box<dyn Error>` работает.

---

## 10. Ментальная модель (самая полезная)

Запомни три правила:

### 🔹 Правило 1

> **Всё, что лежит в стеке — `Sized`**

### 🔹 Правило 2

> **Unsized типы живут только за указателем**

### 🔹 Правило 3

> `T` = Sized
> `T: ?Sized` = «возможно unsized»

---

## 11. Мини-шпаргалка

```rust
T           // Sized (по умолчанию)
T: ?Sized   // может быть unsized

&[u8]       // ok
[u8]        // ❌

&dyn Trait  // ok
dyn Trait   // ❌
```
