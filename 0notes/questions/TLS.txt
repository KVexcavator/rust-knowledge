Окей, TLS — это одна из тех штук, которые **все используют, но мало кто понимает**, пока не копнёшь.
Разберём **медленно, на пальцах и с привязкой к Rust**.

---

# 1. TLS — что это вообще такое

**TLS = Thread-Local Storage**
👉 *поток-локальное хранилище*

Это память, которая:

* **общая для кода**
* **НО отдельная для каждого потока**

То есть:

```text
один и тот же код
разные значения
в каждом потоке
```

---

## Простейшая аналогия

Представь:

* глобальную переменную
* но у каждого потока **своя версия**

```text
thread A: X = 1
thread B: X = 42
```

Они не мешают друг другу.

---

# 2. Зачем TLS вообще нужен

Потому что **глобальные переменные + потоки = боль**.

TLS нужен для:

* errno
* логгеров
* allocator state
* runtime данных
* panic info
* random seed
* tracing spans

В Rust ты используешь TLS **каждый день**, даже не зная.

---

# 3. TLS в железе и ОС (коротко)

На уровне ОС:

* каждому потоку выделяется TLS-сегмент
* CPU знает, где он лежит (через регистры)

Например:

* x86_64 → `FS` / `GS`
* ARM → TPIDR

Доступ:

```asm
mov rax, fs:[offset]
```

→ **очень быстро**

---

# 4. TLS ≠ heap и ≠ stack

Очень важно:

| Хранилище | Lifetime     | Общее? |
| --------- | ------------ | ------ |
| stack     | функция      | нет    |
| heap      | динамический | да     |
| TLS       | поток        | нет    |

TLS живёт:

* **пока жив поток**
* создаётся при старте потока
* уничтожается при его завершении

---

# 5. TLS в Rust: `thread_local!`

Базовый синтаксис:

```rust
use std::cell::RefCell;

thread_local! {
    static COUNTER: RefCell<u32> = RefCell::new(0);
}
```

Использование:

```rust
COUNTER.with(|c| {
    *c.borrow_mut() += 1;
});
```

📌 В каждом потоке будет **свой `COUNTER`**.

---

# 6. Почему почти всегда `RefCell`

Потому что:

* TLS даёт `&'static T`
* но **мутабельность всё равно запрещена**
* нужны правила заимствования → `RefCell`

```rust
thread_local! {
    static BUF: RefCell<Vec<u8>> = RefCell::new(Vec::new());
}
```

---

# 7. Почему нельзя просто `static mut`

```rust
static mut X: i32 = 0; // ❌
```

* общий для всех потоков
* data race
* UB

TLS решает именно это.

---

# 8. Что Rust реально кладёт в TLS

Примеры из std:

* panic payload
* thread id
* RNG
* backtrace state
* allocator cache

👉 Без TLS Rust был бы намного медленнее.

---

# 9. TLS и сигналы (важная связь)

Ты спрашивал раньше про сигналы — вот связь:

❌ **TLS нельзя использовать в signal handler**

Почему:

* TLS использует runtime
* может аллоцировать
* не async-signal-safe

Поэтому signal handlers в Rust — боль.

---

# 10. TLS и async — разные вещи

⚠️ ВАЖНО:

```text
TLS ≠ task-local
```

В async:

* один поток
* много задач
* TLS **разделяется**

Для async есть:

* `tokio::task_local!`

---

# 11. Пример ошибки с TLS

```rust
thread_local! {
    static ID: u32 = get_id();
}
```

❌ если `get_id()` делает что-то сложное
→ может паниковать при старте потока

Лучше:

```rust
thread_local! {
    static ID: RefCell<Option<u32>> = RefCell::new(None);
}
```

---

# 12. Как это выглядит в памяти

```
Process
 ├─ Thread 1
 │   ├─ stack
 │   ├─ TLS → COUNTER=1
 │
 ├─ Thread 2
 │   ├─ stack
 │   ├─ TLS → COUNTER=42
```

---

# 13. Когда TLS — правильный выбор

✔️ данные потока
✔️ кеши
✔️ логгирование
✔️ runtime state

❌ shared state
❌ межпоточное общение
❌ async без task_local

---

# 14. Ментальная модель (запомни)

> **TLS — это `static`, но не `shared`**

---

# 15. Короткая шпаргалка

```rust
thread_local!  // per-thread
static         // global
&'static T     // lifetime, не место хранения
```
