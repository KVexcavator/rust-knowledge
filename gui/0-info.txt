**CUI (Console User Interface)** — это приложения с интерфейсом **в терминале**, но не просто «ввод/вывод текста», а **интерактивные**: окна, списки, меню, таблицы, хоткеи, цвета и т.д.
По сути — **TUI**, просто иногда используют термин *CUI*.

### Как это выглядит

Не GUI (окна мышкой), а примерно как:

* `htop`
* `gitui`
* `lazygit`
* `ncdu`
* `cargo` с интерактивными режимами

---

## CUI на Rust — что это значит

На Rust CUI делают через библиотеки, которые рисуют интерфейс в терминале:

### Основные библиотеки

* **ratatui** (бывш. tui-rs) — ⭐ де-факто стандарт
* **crossterm** — управление клавиатурой, экраном, цветами
* **termion** — альтернатива, больше unix-style
* **dialoguer** — простые интерактивные промпты
* **clap** — CLI, часто используют вместе с CUI

Обычно связка такая:

```
ratatui + crossterm
```

---

## Стоит ли связываться на Ubuntu?

**Коротко: да, на Ubuntu это идеальная среда.**

### Почему именно Ubuntu / Linux

✅ терминал — родная среда
✅ всё стабильно работает (PTY, raw mode, signals)
✅ удобно дебажить
✅ легко деплоить (ssh, tmux, docker)
✅ пользователи dev-инструментов любят CUI

На Windows с этим больше боли, на macOS — норм, но Linux всё равно топ.

---

## Когда CUI — хороший выбор

Тебе **точно стоит**, если ты:

* делаешь **dev tool / админку / утилиту**
* хочешь **быстрое приложение без GUI**
* работаешь по SSH
* ценишь скорость, скриптуемость
* хочешь прокачать Rust (ownership + async + event loop)

Примеры:

* мониторинг
* клиент к API
* админка сервиса
* git / docker / k8s утилита
* локальный менеджер данных

---

## Когда НЕ стоит

❌ если нужен обычный пользователь
❌ если важна мышь / мобильные устройства
❌ если UI сложнее, чем таблицы и формы
❌ если проект — коммерческий массовый продукт

## Порог входа в Rust CUI

Честно:

* **выше, чем в Python**
* но **офигенно окупается**

Ты уже пишешь на Rust → **CUI — логичное продолжение**.

## Минимальный пример (ratatui)

```rust
use ratatui::{Terminal, backend::CrosstermBackend};
use crossterm::{execute, terminal::{enable_raw_mode, disable_raw_mode}};
use std::io::stdout;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    enable_raw_mode()?;
    let mut terminal = Terminal::new(CrosstermBackend::new(stdout()))?;

    terminal.draw(|f| {
        let size = f.size();
        let block = ratatui::widgets::Block::default()
            .title("Hello CUI")
            .borders(ratatui::widgets::Borders::ALL);
        f.render_widget(block, size);
    })?;

    disable_raw_mode()?;
    Ok(())
}
```
