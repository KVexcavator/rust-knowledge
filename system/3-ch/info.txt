В системном программировании одним из основных требований является управление системными ресурсами, такими как память, файлы, сетевой ввод-вывод, устройства и процессы. Каждая операционная система имеет ядро (kernel), представляющее собой центральный программный модуль, загружаемый в память и связывающий аппаратное обеспечение системы с прикладными процессами.
Стандартная библиотека Rust предлагает API-интерфейс для выполнения системных вызовов из программ Rust, чтобы управлять различными системными ресурсами и манипулировать ими.
---
Kernel: Ядро является центральным компонентом операционной системы, который управляет системными ресурсами, такими как память, диск и файловые системы, процессор, сеть и другие устройства, такие как мышь, клавиатура и мониторы. Пользовательские программы (например, инструмент командной строки или текстовый редактор) не могут напрямую управлять системными ресурсами. Им приходится полагаться на ядро ​​для выполнения операций. 
---
Интерфейс системных вызовов (syscall): ядро ​​также предоставляет интерфейс прикладного программирования системных вызовов, который служит точкой входа для процессов, запрашивающих у ядра выполнение различных задач.
---
API-интерфейсы системных вызовов: для выполнения системных вызовов в ядре требуется архитектурно-специфичный код на языке ассемблера. Такой код доступен через библиотеки-обёртки, которые являются платформенно-зависимыми. Для систем Unix/Linux/POSIX такой библиотекой является libc (или glibc). Для операционной системы Windows существуют эквивалентные API.
---
Стандартная библиотека Rust является основным интерфейсом для внедрения программ Rust в функции ядра операционной системы. Она использует libc (или другую эквивалентную библиотеку, специфичную для конкретной платформы) для вызова системных вызовов. Стандартная библиотека Rust является кроссплатформенной, что означает, что детали вызова системных вызовов (или используемые библиотеки-обёртки) не зависят от разработчика Rust. 
===
Rust Standard Library
---
Особенности std
- std является кроссплатформенным
- std доступен для всех пакетов Rust по умолчанию. Например, оператор use std::fs предоставляет доступ к модулю, предоставляющему операции по работе с файлами.
- std включает операции над стандартными примитивами Rust. Например, std::i8::MAX — определяет максимальное значение, которое может быть сохранено в переменной типа i8.
- std реализует основные типы данных, такие как векторы, строки и интеллектуальные указатели, такие как Box, Rc и Arc
- std предоставляет функциональные возможности для таких операций, как манипулирование данными, выделение памяти, обработка ошибок, сетевые функции, ввод-вывод, параллелизм, асинхронные примитивы ввода-вывода и интерфейсы внешних функций
---
Oрганизация std:
- Rust language primitives содержат базовые типы, такие как знаковые и беззнаковые, целые числа, логические числа, числа с плавающей точкой, символьные числа, массивы, кортежи, срезы и строки. 
- core crate — предоставляет типы, трейты, константы и функции, реализованные на основе примитивов Rust, и предоставляет базовые строительные блоки для всего кода Rust. Основной контейнер может использоваться независимо, не привязан к конкретной платформе и не имеет ссылок на библиотеки операционной системы (например, libc) или другие внешние зависимости. 
- alloc crate содержит типы, функции и трейты, связанные с выделением памяти
для значений, выделенных в куче. Он включает в себя типы интеллектуальных указателей, такие как Box (Box<T>), указатели с подсчётом ссылок (Rc<T>) и атомарно подсчётом ссылок (Arc<T>). Он также включает в себя коллекции, такие как Vec и String.
- Модули (библиотеки), которые являются непосредственно частью стандартной библиотеки (и не реэкспортируются из основных или выделенных ящиков), включают в себя богатый функционал для операций, связанных с параллелизмом, вводом-выводом, доступом к файловой системе, сетевыми функциями, асинхронным вводом-выводом, ошибками и специфичными для ОС функциями.
===
Виды стандартных модулей:
- Syscalls-oriented: Это модули, которые либо напрямую управляют аппаратными ресурсами системы, либо требуют наличия ядра для других привилегированных операций.
- Computation-oriented:Это модули, ориентированные на представление данных, вычисления и инструкции для компилятора.
