alloc: содержит API для выделения и освобождения памяти, а также для регистрации пользовательского или стороннего распределителя памяти в качестве распределителя по умолчанию стандартной библиотеки
---
borrow: В Rust принято использовать различные представления заданного типа для различных случаев использования. Например, значение может храниться и управляться как Box<T>, Rc<T> или Arc<T>. Аналогично, строковое значение может храниться как тип String или str. Rust предоставляет методы, позволяющие заимствовать один тип в качестве другого, реализовав метод borrow из трейта Borrow. Таким образом, тип может быть свободно заимствован как множество различных типов. Этот модуль содержит трейт Borrow, который позволяет преобразовывать принадлежащее значение в заимствованное или преобразовывать
заимствованные данные любого типа в принадлежащее значение. Например, значение типа String (являющееся принадлежащим типом) может быть заимствовано как str
---
cell: В Rust безопасность памяти основана на правиле, согласно которому значение может иметь либо несколько неизменяемых ссылок на него, либо одну изменяемую ссылку. Но могут быть ситуации, когда требуется общая изменяемая ссылка. Этот модуль предоставляет совместно используемые изменяемые контейнеры, включая Cell и RefCell. Эти типы  обеспечивают контролируемую изменяемость общих типов
---
clone: ​​В Rust примитивные типы, такие как целые числа, копируются, то есть реализуют трейт Copy. Это означает, что при присвоении значения переменной другой переменной или при передаче параметра функции значение объекта дублируется. Но не все типы можно копировать, поскольку они могут требовать выделения памяти (например, типы String или Vec, где память выделяется в куче, а не в стеке). В таких случаях для дублирования значения используется метод clone(). Этот модуль предоставляет трейт Clone, который  позволяет дублировать значения пользовательских типов данных
---
convert: Этот модуль содержит функционал для упрощения преобразования между типами данных. Например, реализовав трейт AsRef, содержащийся в этом модуле, вы можете написать функцию, принимающую параметр типа AsRef<str>, что означает, что эта функция может принимать любую ссылку, которую можно преобразовать в строковую ссылку (&str). Поскольку оба типа str и String реализуют трейт AsRef, можно передать этой функции либо ссылку на строку (String), либо ссылку на срез строки (&str)
---
default: Этот модуль имеет трейт Default, который используется для назначения осмысленных
значений по умолчанию для типов данных
---
mem: Этот модуль содержит функции, связанные с памятью, включая запрос размера памяти, нициализацию, подкачку и другие операции с памятью
---
pin: Типы в Rust по умолчанию являются перемещаемыми. Например, для типа Vec операция pop() выводит значение, а операция push может привести к перераспределению памяти. Однако существуют ситуации, когда полезно иметь объекты, которые имеют фиксированные области памяти и не перемещаются. Например, самоссылающиеся структуры данных, такие как связные списки. Для таких случаев Rust предоставляет тип данных, который закрепляет данные в определённом месте в памяти. Это достигается путём обёртывания типа в закреплённый
указатель Pin<P>, который закрепляет значение P в памяти
---
ptr: Rust позволяет работать с необработанными указателями в небезопасных блоках кода, где компилятор не несёт ответственности за безопасность памяти, а программист отвечает за безопасные операции с памятью. Этот модуль предоставляет функции для работы с необработанными указателями. Rust поддерживает два типа необработанных указателей — неизменяемые (например, *const i32) и изменяемые (например, *mut i32). Необработанные указатели не имеют ограничений на использование. Это единственный тип указателей в Rust, который может быть равен null, и автоматического разыменования необработанных указателей не предусмотрено
---
rc: предоставляет однопоточные указатели с подсчётом ссылок, где rc означает «с подсчётом ссылок». Указатель с подсчётом ссылок на объект типа T можно представить как Rc<T>. Rc<T> обеспечивает совместное владение значением T, которое выделяется в куче. Если значение этого типа клонируется, возвращается новый указатель на тот же участок памяти в куче (не дублирует значение в памяти). Это значение сохраняется до тех пор, пока не появится последний указатель Rc, ссылающийся на это значение, после чего значение удаляется
