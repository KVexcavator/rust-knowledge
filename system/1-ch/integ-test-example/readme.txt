Для написания интеграционных тестов в Rust принято создавать каталог tests в корневой папке пакета и создавать в этом каталоге один или несколько файлов, каждый из которых содержит один интеграционный тест. Каждый файл в каталоге tests рассматривается как отдельный контейнер.
---
cargo new --lib integ-test-example && cd integ-test-example
---
все тесты
cargo test
каждый по отдельности
test files_test1
test files_test2
все начинающееся с files_test
cargo test files
запустить толко игнорируемые
cargo test -- --ignored
---
Последовательный или параллельный запуск тестов
По умолчанию команда cargo test запускает различные тесты параллельно в отдельных потоках. 
Для поддержки этого режима выполнения тестовые функции должны быть написаны таким образом, чтобы не было совместного использования общих данных между тестовыми случаями. Однако, если такая необходимость действительно существует (например, один тестовый случай записывает данные в определённое место, а другой их считывает), то мы можем запустить тесты последовательно следующим образом:
cargo test -- --test-threads=1
Эта команда указывает команде cargo использовать только один поток для выполнения тестов, что косвенно означает, что тесты должны выполняться последовательно.
