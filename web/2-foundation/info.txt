Вызов функции:
impl Question {
  fn new(id: QuestionId, title: String, …) -> Self {
    Question {
      id,
      title,
      content,
      tags,  
    }
  }

  fn update_title(&self, new_title: String) -> Self {
    Question::new(self.id, new_title, self.content, self.tags)
  }
}
--
Связанная функция (new) не принимает параметр &self и вызывается с помощью двойного двоеточия (::):
let q = Question::new(QuestionId("1".to_string()), "title".to_string(), …);
---
Метод (update_title) принимает параметр &self и вызывается с помощью точечной нотации (.):
q.update_title("better_title".to_string());
===
Главное отличие между String и &str в Rust заключается в том, что размер строки можно изменять. Строка — это коллекция байтов, реализованная в виде вектора:
pub struct String {
  vec: Vec<u8>,
}
Строки создаются с помощью String::from("popcorn"); и могут быть изменены после создания.
---
&str указатель на область памяти, разрешено читать её, но не изменять. Это делает использование &str более эффективным с точки зрения использования памяти.
----
Вот практическое правило: 
- При создании функций, используют &str как тип параметра для чтения сроки
- Если нужно владеть и\или изменять строку, используют String
===
